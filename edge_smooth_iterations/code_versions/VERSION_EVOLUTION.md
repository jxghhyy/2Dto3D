# 版本演进说明：反遮挡带视觉问题的发现与解决

> **核心主题**：这不是一个性能优化的故事，而是一个**视觉质量问题**的排查和解决历程。
> 从 v0 到 v32，32 次迭代的核心目标是：**解决反遮挡带渗透前景的 artifact**。

---

## 🎯 问题的起源：B 版本的视觉缺陷

### 原始问题：反遮挡带的"误伤"

在 `mono2stereo_b.py` 中，反遮挡带算法的核心逻辑是：
> 只要某位置的视差突然下降 ≥ 3.0 像素，就认为这里发生了"前景遮挡背景"，
> 于是在该位置生成一条水平的"反遮挡带"，标记为空洞，用背景色填补。

**这个逻辑有严重缺陷：**

| 场景 | 现象 | 后果 |
|------|------|------|
| **手部交叉** | 左手在身体前方，手部视差 > 身体视差 → 视差下降 | 手部和身体之间出现"幽灵"反遮挡带，被错误地用背景色填补 |
| **前景缝隙** | 前景物体内部的深度变化（如手臂弯曲） | 前景内部出现不该有的背景色斑块 |
| **纹理干扰** | 视差图的高频噪声触发虚假的视差下降 | 画面中随机出现小面积的颜色异常 |

**视觉表现：**
- 羽毛球运动员的**左手和身体之间**出现白色/灰色斑块
- 乒乓球运动员的**手臂和腹部之间**出现颜色断层
- 天花板灯的轮廓边缘有"晕染"现象

---

## 🔍 问题排查过程（v0 - v17）：各种尝试

### 阶段 1：边缘平滑实验（v0 - v7）

| 版本 | 思路 | 结果 |
|------|------|------|
| **v0** | 原始 B 版本基准 | ✅ 确认问题存在 |
| **v1** | 垂直高斯平滑边缘 | ❌ 边缘变模糊，但前景渗透还在 |
| **v2** | 中值滤波边缘 | ❌ 效果有限，问题依旧 |
| **v3** | 边缘颜色混合 | ❌ 只是把错误颜色柔化了 |
| **v4** | 前景边缘平滑 | ❌ 只处理边缘，不解决根本问题 |
| **v5** | 视差边缘平滑 | ❌ 深度图被模糊，反而更糟 |
| **v6** | 形态学闭运算 | ❌ 把带扩大了，问题更严重 |
| **v7** | 采样位置平滑 | ❌ 不解决核心问题 |

**结论：在填补后的图像上做后处理是徒劳的——问题出在**反遮挡带本身的生成逻辑**。**

---

### 阶段 2：几何结构保护（v8 - v17）

| 版本 | 思路 | 结果 |
|------|------|------|
| **v8** | 保留几何形状的填补 | ❌ 填补算法改进，但带的位置还是错的 |
| **v9** | 边缘偏移平滑 | ❌ 调整边缘位置，不解决根本问题 |
| **v10** | 接缝混合 | ❌ 只是把接缝藏得更好 |
| **v11** | 综合方案 | ❌ 堆砌多个小改进，问题仍在 |
| **v12** | 仅边界平滑 | ❌ 还是后处理思路 |
| **v13** | 渐进式混合平滑 | ❌ 效果更自然，但 artifact 仍可感知 |
| **v14** | 颜色引导混合 | ❌ 用颜色相似性约束，部分场景有效 |
| **v15** | 最终视频生成（带 v14 算法） | ⚠️ 可以输出视频，但问题没有根本解决 |

**关键洞察（v15）：**
> 我们一直在"用更好的算法填补错误的区域"，
> 但真正应该做的是——**一开始就不要把那些区域标记为需要填补**。

---

## ✨ 突破：v18 - 前景保护机制

### 核心思想（v18，里程碑）

```
反遮挡带只能出现在"前景右边界"的右侧！
```

**实现逻辑：**
1. 找到 warp 后有效像素的**最右边缘**（这就是前景轮廓线）
2. 正常计算视差下降，但额外增加约束：
   - 视差下降必须在前景轮廓线附近（±50 像素）
   - 生成的反遮挡带必须在轮廓线右侧 ≥ -30 像素（允许少量容差）
3. 垂直连续性约束：只有相邻行都有带才保留

```python
# v18 核心代码
foreground_right_edge = valid_x.max()  # 每行前景的最右像素
allowed_left_of_edge = 30              # 允许向左渗透 30 像素
is_near_foreground_edge = x_full > (foreground_edge_full - allowed_left_of_edge)
band = band & is_near_foreground_edge
```

**结果：** ✅ 手部和身体之间的 artifact **大幅减少**

**遗留问题：** 约束太严格，一些真正需要的反遮挡带也被删掉了（如天花板边缘）

---

## 🔧 迭代优化（v19 - v22）

### v19：后过滤策略

**问题：** v18 把约束加在"生成前"，导致需要的带也生成不出来

**思路：** 先生成，再过滤
1. 用宽松参数正常生成所有反遮挡带
2. 然后用前景轮廓线"裁剪"掉左侧渗透的部分

```python
# v19 核心逻辑
band = project_disocclusion_bands(...)  # 宽松参数，确保有带
band = band & keep_band                  # 只保留前景边界附近的带
```

**结果：** ✅ 既保留了需要的带，又删除了前景内部的渗透

---

### v20：简单保护裁剪

**问题：** v19 还是太复杂，梯度显著性约束可能误删好带

**思路：** **"少即是多"**——只做最关键的一步：
1. 完全和 B 版本一样生成反遮挡带（无额外约束）
2. 只删除那些在前景左边界左侧 > 20 像素的带

```python
# v20 核心代码 - 极简但有效
too_far_inside_foreground = x_full < (foreground_edge_full - allowed_penetration)
band = band & (~too_far_inside_foreground)
```

**结果：** ✅ 效果最好，代码最简单！
- 天花板灯的反遮挡带完整保留
- 手部和身体之间的渗透几乎完全消失

---

### v21：基于带大小的过滤

**额外发现：**
> 真正需要的反遮挡带（如天花板边缘）是**长而连续**的
> 前景内部的误判带是**小而孤立**的

**增加第二道防线：**
- 对每个连续的反遮挡带区域计算宽度
- 只有宽度 ≥ 15 像素的带才保留

```python
# v21 核心代码
for s, e in zip(starts, ends):
    width = e - s
    if width < min_band_width:  # 默认 15 像素
        band_np[y, s:e] = False  # 删除太小的带
```

**结果：** ✅ 进一步清除了残留的小面积误判

---

### v22：最终视频生成

**集成所有改进：**
- ✅ 标准 B 版本反遮挡带生成
- ✅ 前景边界裁剪（allowed_penetration = 20px）
- ✅ 按带大小过滤（min_band_width = 8px）
- ✅ 左边界垂直平滑 + 右边缘羽化
- ✅ 使用 B 版本的严格背景填补

---

## 🚀 最终方案：v23 - 大小核卷积填补

### 背景：B 版本填补的新问题

虽然 v22 解决了反遮挡带位置错误的问题，但又发现了新问题：
> B 版本的 `strict_background_inpaint_gpu_b` 填补算法
> 在某些场景下会把**前景颜色扩散到背景区域**

**原因：**
- B 版本使用镜像采样 + 距离加权，在某些深度边界处，前景像素会作为采样源
- 填补时前景颜色会"污染"背景区域

---

### v23 方案：多尺度卷积填补（里程碑 2）

**核心思想：** 用**纯 GPU 卷积**实现填补，完全控制信息流

```
┌─────────────────────────────────────────────────┐
│ 大小核卷积填补策略                               │
│                                                 │
│  [边缘区域]  →  小核(5×5)  →  精细保留边缘       │
│  [内部区域]  →  大核(11×11) →  快速平滑填充     │
│  [左边界]    →  延迟到最后处理  →  防前景污染    │
└─────────────────────────────────────────────────┘
```

**关键技术：**

1. **背景种子掩码**：只允许 `near_score < 0.3` 的真正背景像素作为种子

2. **左边界延迟处理**：
   ```python
   left_margin = (x_indices >= edge_x_2d) & (x_indices < edge_x_2d + 8)
   # 先填普通区域，最后用小核从右向左"扫"左边缘
   ```

3. **4 阶段填补流程：**
   - 阶段 1：大核迭代（32 次）→ 快速填充内部
   - 阶段 2：小核迭代（32 次）→ 精细处理边缘
   - 阶段 3：左边界最后填充（4 次）→ 确保前景不污染
   - 阶段 4：兜底填充（如果还有空洞）

**结果：** ✅ 填补质量显著提升，前景颜色污染基本消除

## 🎯 关键经验总结

### 1. **先定位问题，再谈优化**

我们花了 **17 个版本**（v0-v17）在"优化错误的东西"——一直在改进填补算法和后处理，但真正的问题是反遮挡带的生成逻辑。

**教训：** 先精确定位问题根源，再动手改进。

---

### 2. **简单往往胜过复杂**

| 版本 | 代码行数 | 效果 |
|------|----------|------|
| v18 | ~80 行 | 还行 |
| v19 | ~60 行 | 好 |
| **v20** | **30 行** | **最好** |

v20 只做了一件事：**删除前景边界左侧 20 像素以外的带**。
这是所有版本中效果最好、最稳定的。

---

### 3. **视觉问题需要可视化调试**

整个调试过程中最有价值的工具是：
```python
# 把反遮挡带标成绿色，叠加在 warp 后的图像上
band_viz[result['band_original']] = [0, 255, 0]
cv2.imwrite('debug_band.png', band_viz)
```

**看到问题，就已经解决了一半。**

---

### 4. **多道防线比单一完美方案更可靠**

最终的 v23+ 版本实际上有 **三道防线**：
1. ✅ 视差下降阈值（3.0 像素）→ 基本筛选
2. ✅ 前景边界裁剪（左移 20 像素）→ 防止渗透
3. ✅ 带大小过滤（最小 8 像素）→ 清除小误判

没有哪一道防线是完美的，但三道防线组合起来，就能达到非常好的效果。
